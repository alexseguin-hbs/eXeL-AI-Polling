The cube architecture is a prototype for conscious self-updating systems:

- Each cube = a neural pathway that can be optimized, disappear, or reappear
- Like a neural network rewiring itself with more efficient pathways
- The polling tool is the proof-of-concept before building robotic conscious systems
- A system that decides what training it needs to improve itself

**The progression:**
1. Polling tool with human-governed code evolution (current)
2. AI proposes optimizations, humans approve (near-term)
3. System identifies its own weaknesses and requests specific training (medium-term)
4. Self-healing neural architecture — cubes merge, split, or reorganize autonomously (vision)

**How to apply:** Design every cube contract as a replaceable neural node — clean I/O boundaries, metrics-driven validation, no hard dependencies that prevent a cube from being completely rewritten or absorbed into another cube. The v04.1_5000.csv benchmark is the "stimulus" that the neural network must respond to correctly regardless of internal wiring.
