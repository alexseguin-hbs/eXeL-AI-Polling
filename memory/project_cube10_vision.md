Cube 10 Code Challenge is simulation-only for now using v04.1_5000.csv (5000 responses) as the test dataset.

**Vision:** The system will eventually update its own code by:
1. Aggregating human prioritization votes (Cube 7 ranking)
2. AI generates code improvement proposals based on voted priorities
3. ADMIN team approves/rejects AI recommendations
4. Approved changes auto-deploy (replacing manual GitHub workflow)

**Why:** The goal is a self-healing, self-improving governance engine where human intent (votes) drives AI code evolution under human oversight (ADMIN approval gate).

**How to apply:** Every Cube checkout contract must show full I/O + CRS + sub-CRS tasks so that both humans AND AI can understand the contract well enough to propose improvements. The v04.1_5000.csv dataset is the universal benchmark — any code change must produce equal or better metrics against this dataset.
